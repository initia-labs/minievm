# Contributing to Minievm

Thank you for your interest in contributing to Minievm! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your changes
4. Make your changes
5. Submit a pull request

## Pull Request Process

Before submitting a pull request, please ensure:

- Your code follows the project's style guidelines
- All tests pass
- Your commit messages are clear and descriptive
- You have added tests for new functionality

## Reporting Issues

If you find a bug or have a suggestion for improvement, please open an issue on GitHub.

## Code of Conduct

Please be respectful and professional in all interactions with other contributors and maintainers.

## Getting Help

If you have questions about contributing, feel free to open an issue or reach out to the maintainers.

For more details on the pull request process, see [.github/PULL_REQUEST_TEMPLATE.md](./.github/PULL_REQUEST_TEMPLATE.md).
